# 教师讲解提纲

## 1. 课程定位

本项目用于 40 课时实战课程。讲课时不把重点放在某个框架本身，而是放在“一个 AI 应用是怎样被完整做出来的”。

核心表达：

```text
页面负责交互，接口负责连接，业务层负责流程，大模型负责生成，数据库负责保存。
```

## 2. 第一阶段：系统跑起来

讲解内容：

1. 什么是前后端分离；
2. 为什么前端不用直接调用 Ollama；
3. 为什么需要后端；
4. 为什么要用接口传 JSON；
5. 为什么要先写 health 接口。

课堂目标：学生知道系统由前端、后端、模型、数据库四部分组成。

## 3. 第二阶段：前端页面

只讲 React 最小知识：

1. 组件：页面上的一个功能块；
2. useState：保存页面数据；
3. onClick：按钮点击；
4. onChange：输入框变化；
5. fetch：调用后端接口。

不讲 React 底层、Redux、TypeScript、复杂路由、性能优化。

## 4. 第三阶段：后端分层

```text
api          Controller
services     Service
repositories DAO
schemas      DTO
core         Config
utils        工具类
```

重点：接口层不要写复杂业务；业务层不要直接堆 SQL；数据库操作单独放 Repository；请求和响应格式放 Schema。

## 5. 第四阶段：调用 Ollama

讲解顺序：

1. Ollama 在本地运行模型；
2. FastAPI 调用 Ollama 的 HTTP 接口；
3. 前端不直接访问 Ollama；
4. 后端封装 llm_service；
5. 以后换模型只改模型调用层。

## 6. 第五阶段：资料问答 RAG

```text
资料切分 → 根据问题检索相关片段 → 拼接 Prompt → 交给大模型回答
```

先讲关键词检索，不一开始讲复杂向量数据库。

## 7. 第六阶段：题目生成与错题本

讲解重点：

1. Prompt 能控制输出格式；
2. 数据库能保存重要内容；
3. 错题本是 AI 生成内容与传统系统功能的结合。

## 8. 课堂演示顺序

1. 打开首页；
2. 查看模型状态；
3. 普通问答；
4. 上传资料；
5. 开启资料问答；
6. 生成练习题；
7. 保存错题；
8. 查看数据库记录。
