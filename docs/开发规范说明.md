# 开发规范说明

## 1. 后端分层规范

后端采用类 MVC 分层思想：

```text
API 层          接收请求、返回响应
Service 层      处理业务逻辑
Repository 层   操作数据库
Schema 层       定义请求与响应格式
Core 层         配置、数据库初始化
Utils 层        通用工具函数和 Prompt 模板
```

讲解时可以类比 Java Web：

| Python/FastAPI | Java Web | 说明 |
|---|---|---|
| api | Controller | 接收前端请求 |
| services | Service | 处理业务流程 |
| repositories | DAO / Mapper | 操作数据库 |
| schemas | DTO / VO | 定义前后端数据格式 |
| core | Config | 全局配置 |
| utils | Utils | 工具函数 |

## 2. 持久化规范

当前版本涉及三类核心持久化：

1. 智能问答保存到 `chat_messages`；
2. 题目生成和知识总结保存到 `generation_records`；
3. 错题本保存到 `wrong_questions`。

原则：

```text
凡是刷新页面后仍然应该看到的数据，都必须保存到数据库。
```

## 3. 前端展示规范

大模型输出通常是 Markdown 文本。前端不要直接用 `<pre>` 原样显示，否则页面会出现大量 `#`、`**`、`-` 等符号。

当前项目统一使用：

```text
components/MarkdownView.jsx
```

该组件负责把 Markdown 渲染为标题、列表、加粗文本、代码块等更规整的页面结构。

## 4. 代码注释规范

本项目注释以教学为目标：

1. 注释解释“为什么这样写”；
2. 不只解释“这行代码是什么”；
3. 关键分层、数据库、模型调用、RAG、Markdown 渲染位置要写清楚；
4. 注释保持专业、规范，不使用随意口语。
